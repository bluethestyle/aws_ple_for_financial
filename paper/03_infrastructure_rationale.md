# Infrastructure & Deployment Rationale

## 1. 금융사 AI 인프라 현실

### 왜 SageMaker인가 (vs Kubernetes)

| | SageMaker | Kubeflow + K8s |
|--|-----------|----------------|
| GPU 관리 | AWS가 프로비저닝 | 직접 클러스터 구축/운영 |
| 비용 구조 | 쓴 만큼 과금 | 하드웨어 + 운영 인력 (2-3명) |
| 학습 빈도 | 일~주 1회 배치에 적합 | 상시 GPU 수백 장 경쟁에 적합 |
| 운영 부담 | 낮음 | 높음 |
| 적합 대상 | 금융사, 중견 커머스 | 빅테크 (GPU 수백 장 상시 가동) |

### 금융사 vs 빅테크 모델 운영 차이

| | 금융사 | 빅테크 |
|--|--------|--------|
| 모델 수 | 수십 개 | 수천 개 |
| 재학습 주기 | 일~월 1회 | 시간~일 단위 |
| GPU 가동 시간 | 하루 2-3시간 | 상시 (20시간+) |
| 핵심 가치 | 안정적 성능 유지 + 규제 준수 | 성능 극대화 = 매출 |
| 모델 교체 | 금감원 검증 후 (수개월) | 즉시 A/B 테스트 배포 |
| GPU 스케줄링 경쟁 | 거의 없음 | 치열 (우선순위 큐 필수) |

### K8s가 불필요한 이유
- 추론 서버: 모델 로드 후 상시 점유 → 스케줄러 역할 없음
- 학습: 모델 10개 미만, 일 1회 → GPU 경쟁 없음
- 국내 금융사 K8s 구축 사례 (우리은행 등): SI 업체 주도, 실제 학습 수요 대비 과잉 인프라

## 2. 프로덕션 배포 아키텍처

### 서빙 구조
```
[API Gateway]
     ↓
[Lambda Function]
  ├── LGBM 모델 로드 (수 MB, 콜드스타트 ~1s)
  ├── 추론 (ms 단위)
  ├── IG attribution 계산
  ├── 추천사유 템플릿 생성
  └── 응답 반환
```

### 서버리스 선택 이유
- 금융사 추천 요청: 초당 수십~수백 건 (빅테크 대비 매우 낮음)
- 상시 GPU 서버 유지 불필요 (증류된 LGBM은 CPU 추론)
- Lambda 과금 = 요청 수 비례 → 트래픽 낮을 때 비용 0
- 모델 업데이트: Lambda 버전 교체로 무중단 배포

### 데이터 흐름 (온프렘 ↔ 클라우드 하이브리드)
```
온프렘: 계정계, 원장, 고객DB (규제상 이전 불가)
  ↓ 비식별 처리 / 집계 데이터
클라우드 (AWS):
  ├── Phase 0: S3 + SageMaker Processing (데이터 전처리)
  ├── Phase 1-3: SageMaker Training (학습/ablation)
  ├── Phase 4: SageMaker Training (증류)
  └── Phase 5: Lambda + API Gateway (서빙)
```

## 3. GPU 확장 전략

### 현재 (ablation)
- 로컬 RTX 4070 12GB, 순차 48 시나리오 (~24시간)
- 목적: 모델/데이터 검증

### 클라우드 ablation
- SageMaker Spot g5.xlarge (A10G 24GB) × 4대 병렬
- 48 시나리오 / 4대 = 12 시나리오/대 → ~4시간
- 비용: ~$16 (Spot 기준)
- CLAUDE.md 규칙: Spot 동시 4대 이하 (같은 AZ 경쟁 방지)

### 프로덕션 학습 확장 (향후)
- Spark Ingestion → DuckDB 앞단 (수억 row 분산 전처리)
- DDP wrapper → trainer.py (`--multi-gpu` 선택적 활성화)
- 현재 코드에서 이 두 곳만 추가하면 됨 (나머지는 config-driven)

## 4. 비용 효율성 주장

| 방식 | 월 비용 (추정) | 비고 |
|------|---------------|------|
| K8s 자체 구축 (GPU 4장) | $8,000-15,000 | 하드웨어 감가 + 인력 |
| SageMaker 온디맨드 | $500-2,000 | 주 1회 학습 + 서빙 |
| SageMaker Spot + Lambda | $200-800 | 최적화 시 |

금융사 규모에서 SageMaker + Lambda 조합이 K8s 대비 TCO 70-90% 절감 가능.
논문 주장: "금융 도메인 특성상 상시 GPU 클러스터 불필요, 서버리스 아키텍처가 TCO 최적"
